# Incident Response Runbook (AI Systems)

**Objective:** Respond to AI-related incidents quickly and safely.

## Incident Types

- Data exposure or leakage
- Unsafe or incorrect AI output in production
- System outage or severe latency
- Cost spike or resource exhaustion

## Response Steps

1. **Detect and Triage**
   - Identify incident type and severity
   - Assign incident commander

2. **Contain**
   - Disable affected AI workflows
   - Restrict access if needed
   - Preserve logs and evidence

3. **Investigate**
   - Root cause analysis
   - Scope of impact
   - Data exposure assessment

4. **Recover**
   - Restore services safely
   - Validate outputs and tests
   - Monitor for recurrence

5. **Postmortem**
   - Document findings and fixes
   - Update policies and controls
   - Share lessons learned
